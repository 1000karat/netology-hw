#### Требования к результатам 
- [ ] Прикрепите к файлу README.md скриншот systemctl status prometheus, где будет написано: prometheus.service — Prometheus Service Netology Lesson 9.4 — [Ваши ФИО]

#### 🔧 Пошаговый процесс
1.  Создаём пользователя для Prometheus:
    ``` bash
    useradd --no-create-home --shell /bin/false prometheus
    ```
2. Качаем последний релиз из репозитория [prometheus](https://github.com/prometheus/prometheus): 
	``` bash
    wget https://github.com/prometheus/prometheus/releases/download/v3.9.1/prometheus-3.9.1.linux-amd64.tar.gz
    ```
3. Распаковываем и переходим в каталог:
    ``` bash
    tar vxfz prometheus*.tar.gz
	cd prometheus*/
    ```
4. Создаём необходимые каталоги, переносим файлы и назначаем права:
    ``` bash
    mkdir /etc/prometheus
	mkdir /var/lib/prometheus
	mv promtool /usr/local/bin
	mv prometheus /usr/local/bin
	mv prometheus.yml /etc/prometheus/
	chown -R prometheus:prometheus /usr/local/bin/prometheus
	chown -R prometheus:prometheus /usr/local/bin/promtool
	chown -R prometheus:prometheus /var/lib/prometheus
	chown -R prometheus:prometheus /etc/prometheus
	chown -R prometheus:prometheus /etc/prometheus/prometheus.yml
    ```
5.  Создаём службу для Prometheus:
    ``` bash
    vim /etc/systemd/system/prometheus.service
    ```
	содержимое:
    ``` bash
    [Unit]
	Description=Prometheus Service Netology Lesson 9.4 [sysoevav]
	Wants=network.target
	After=network.target

	[Service]
	User=prometheus
	Group=prometheus
	Type=simple

	ExecStart=/usr/local/bin/prometheus \
		--config.file=/etc/prometheus/prometheus.yml \
		--storage.tsdb.path=/var/lib/prometheus/

	Restart=always
	RestartSec=5

	[Install]
	WantedBy=multi-user.target
    ```
6. Изменяем localhost на IP в конфигурации по пути /etc/prometheus/prometheus.yml:
    ``` yml
	- targets: ["192.168.18.142:9090"]
	```
6. Перезагрузка systemd и запуск службы
    ``` bash
    systemctl daemon-reload
	systemctl enable prometheus
	systemctl start prometheus
    ```
7. Проверка статуса:
    ``` bash
    systemctl status prometheus
    ```

#### Требования к результатам
- [ ] Прикрепите к файлу README.md скриншот systemctl status node-exporter, где будет написано: node-exporter.service — Node Exporter Netology Lesson 9.4 — [Ваши ФИО]

#### 🔧 Пошаговый процесс
1. Качаем последний релиз из репозитория [node_exporter](https://github.com/prometheus/node_exporter): 
	``` bash
    wget https://github.com/prometheus/node_exporter/releases/download/v1.10.2/node_exporter-1.10.2.linux-amd64.tar.gz
    ```
2. Распаковываем и переходим в каталог:
    ``` bash
    tar xvfz node_exporter-1.10.2.linux-amd64.tar.gz
	cd node*/
    ```
3. Создаём необходимые каталоги, переносим файлы и назначаем права:
    ``` bash
    mkdir /etc/node_exporter
	mv node_exporter /etc/node_exporter
	chown -R prometheus:prometheus /etc/node_exporter
    ```
4.  Создаём службу для Prometheus:
    ``` bash
    vim /etc/systemd/system/node-exporter.service
    ```
	содержимое:
    ``` bash
	[Unit]
	Description=Node Exporter Lesson 9.4 [sysoevav]
	Wants=network.target
	After=network.target

	[Service]
	User=prometheus
	Group=prometheus
	Type=simple

	ExecStart=/etc/node_exporter/node_exporter

	Restart=always
	RestartSec=5

	[Install]
	WantedBy=multi-user.target
    ```
5. Добавляем в конфигурацию /etc/prometheus/prometheus.yml строки:
    ``` yml
    - job_name: "node-exporter"
    static_configs:
      - targets: ["192.168.18.142:9100"]
        labels:
          app: "node-exporter"
    ```
6. Перезагрузка systemd и запуск службы
    ``` bash
	systemctl daemon-reload
	systemctl enable node-exporter
	systemctl start node-exporter
    ```
7. Проверка статуса:
    ``` bash
    systemctl status node-exporter
    ```
