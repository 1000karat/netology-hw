### Цели задания
1. Настраивать регулярные задачи на резервное копирование (полная зеркальная копия)
2. Настраивать инкрементное резервное копирование с помощью rsync

### Задание 1 
- Составьте команду rsync, которая позволяет создавать зеркальную копию домашней директории пользователя в директорию `/tmp/backup`
- Необходимо исключить из синхронизации все директории, начинающиеся с точки (скрытые)
- Необходимо сделать так, чтобы rsync подсчитывал хэш-суммы для всех файлов, даже если их время модификации и размер идентичны в источнике и приемнике.

#### Требования к результатам 
- На проверку направить скриншот с командой и результатом ее выполнения

#### 🔧 Пошаговый процесс
1. Составьте команду rsync, которая позволяет создавать зеркальную копию домашней директории пользователя в директорию `/tmp/backup`
    ``` bash
	mkdir /tmp/backup
    rsync -avh --delete $HOME/ /tmp/backup
    ```
2. 📌 Скриншот:
![Скриншот](https://raw.githubusercontent.com/1000karat/netology-hw/refs/heads/main/img/hw-07-01.png) 

3. Необходимо исключить из синхронизации все директории, начинающиеся с точки (скрытые)
    ``` bash
	rsync -avh --delete --exclude '.*' $HOME/ /tmp/backup
    ```
4. 📌 Скриншот:
![Скриншот](https://raw.githubusercontent.com/1000karat/netology-hw/refs/heads/main/img/hw-07-02.png) 

5. Необходимо сделать так, чтобы rsync подсчитывал хэш-суммы для всех файлов, даже если их время модификации и размер идентичны в источнике и приемнике
    ``` bash
	rsync -avc --delete --exclude='.*' $HOME/ /tmp/backup/
    ```
6. 📌 Скриншот:
![Скриншот](https://raw.githubusercontent.com/1000karat/netology-hw/refs/heads/main/img/hw-07-03.png) 

### Задание 2 
- Написать скрипт и настроить задачу на регулярное резервное копирование домашней директории пользователя с помощью rsync и cron.
- Резервная копия должна быть полностью зеркальной
- Резервная копия должна создаваться раз в день, в системном логе должна появляться запись об успешном или неуспешном выполнении операции
- Резервная копия размещается локально, в директории `/tmp/backup`

#### Требования к результатам
- На проверку направить файл crontab и скриншот с результатом работы утилиты.

#### 🔧 Пошаговый процесс
1. Файл [backup_home_dir.sh](https://github.com/1000karat/netology-hw/blob/main/files/backup_home_dir.sh) размещается по пути: `/usr/local/bin/`.
2. Выполняется chmod на Execution:
    ``` bash
	sudo chmod +x /usr/local/bin/backup_home_dir.sh
	```
2. В `crontab -e` дописывается строка:
    ``` bash
	1 2 * * * /usr/local/bin/backup_home_dir.sh user
    ```
3. Команда для journalctl:
    ``` bash
	journalctl --since "2026-03-07 17:40" --until "2026-03-08" | grep "backup_home_dir"
    ```
4. 📌 Скриншот:
