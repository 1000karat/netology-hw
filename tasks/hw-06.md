### Цели задания
1. Настраивать балансировку с помощью HAProxy
2. Настраивать связку HAProxy + Nginx

### Задание 1 
1. Запустите два simple python сервера на своей виртуальной машине на разных портах
2. Установите и настройте HAProxy, воспользуйтесь материалами к лекции по ссылке
3. Настройте балансировку Round-robin на 4 уровне.

#### Требования к результатам 
На проверку направьте конфигурационный файл haproxy, скриншоты, где видно перенаправление запросов на разные серверы при обращении к HAProxy.

#### 🔧 Пошаговый процесс
1.  В конец дефолтного конфигурационного файла [haproxy.cfg](https://github.com/1000karat/netology-hw/blob/main/files/haproxy_01.cfg) по пути `/etc/haproxy/` дописываются строки:
    ``` bash
    # балансировка
	# http = 7; tcp = 4
	frontend front
        bind *:8088
        mode tcp
        default_backend balancer_back

	# httpchk = 7; tcp-check = 4
	backend balancer_back
        mode tcp
        balance roundrobin
        option tcp-check
	#   http-check send meth GET uri /index.html
        server srv1 127.0.0.1:8888 check
        server srv2 127.0.0.1:9999 check

	# Страница статистики
	listen stats
        bind *:8404
        mode http
        stats enable
        stats uri /stats
        stats refresh 5s
    ```
2. Проверка файла /etc/haproxy/haproxy.cfg
    ``` bash
    sudo haproxy -c -f /etc/haproxy/haproxy.cfg
    ```
3. Перезапуск сервиса
    ``` bash
    sudo systemctl reload haproxy.service
    ```

### Задание 2 
1. Запустите три simple python сервера на своей виртуальной машине на разных портах
2. Настройте балансировку Weighted Round Robin на 7 уровне, чтобы первый сервер имел вес 2, второй - 3, а третий - 4
HAproxy должен балансировать только тот http-трафик, который адресован домену example.local

#### Требования к результатам
На проверку направьте конфигурационный файл haproxy, скриншоты, где видно перенаправление запросов на разные серверы при обращении к HAProxy c использованием домена example.local и без него.

#### 🔧 Пошаговый процесс
1. В файл hosts по пути /etc/hosts дописывается строка:
    ``` bash
	127.0.0.1 example.local
	```
2. В конец дефолтного конфигурационного файла [haproxy.cfg](https://github.com/1000karat/netology-hw/blob/main/files/haproxy_02.cfg) по пути `/etc/haproxy/` дописываются строки:
    ``` bash
	frontend front
        bind *:80
        mode http
        acl host_example hdr(host) -i example.local
        use_backend example_balancer if host_example
        default_backend default

	backend example_balancer
        mode http
        balance roundrobin
        option httpchk
        http-check send meth GET uri /index.html
        server srv1 127.0.0.1:8881 weight 2 check
        server srv2 127.0.0.1:8882 weight 3 check
        server srv3 127.0.0.1:8883 weight 4 check

	backend default
        mode http
        http-request deny

	listen stats
        bind *:8404
        mode http
        stats enable
        stats uri /stats
        stats refresh 5s
    ```
3. Проверка файла /etc/haproxy/haproxy.cfg
    ``` bash
    sudo haproxy -c -f /etc/haproxy/haproxy.cfg
    ```
4. Перезапуск сервиса
    ``` bash
    sudo systemctl reload haproxy.service
    ```
5. Команда проверки:
    ``` bash
    for i in {1..25}; do curl -s http://example.local; done
    ```
